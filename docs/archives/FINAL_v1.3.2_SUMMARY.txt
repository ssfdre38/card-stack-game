🔧 FIXED: CORNER NUMBERS NOW VISIBLE

Problem in v1.3.1:
❌ Corner numbers weren't showing up
❌ Over-complicated boundary calculations
❌ Too conservative positioning

Solution in v1.3.2:
✅ Simplified positioning logic
✅ Better text sizing (18sp small, 26sp large)
✅ Clear white text with bold styling
✅ Proper anti-aliasing
✅ Safe margins (15-20px from edges)

Technical Fix:
```java
// Simple and effective
Paint cornerPaint = new Paint(Paint.ANTI_ALIAS_FLAG);
cornerPaint.setColor(Color.WHITE);
cornerPaint.setTextSize(isSmall ? 18 : 26);
cornerPaint.setTextAlign(Paint.Align.CENTER);
cornerPaint.setFakeBoldText(true);

// Top-left
float topY = 20 + cornerPaint.getTextSize();
float leftX = 15 + cornerTextWidth / 2;

// Bottom-right
float bottomY = height - 20;
float rightX = width - 15 - cornerTextWidth / 2;
```

Result:
✅ Corner numbers perfectly visible
✅ Proper positioning on all card sizes
✅ Clear and readable
✅ Professional appearance
